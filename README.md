# San Diego County Heat-Relief Gap Finder

An interactive map that flags the census tracts most in need of a new cooling or water site. It scores every tract in San Diego County on heat exposure, social vulnerability, population, and distance to existing relief, then ranks where a new site would help the most.

**Live app:** https://heat-gap-finder.streamlit.app/#san-diego-county-heat-relief-gap-finder

## What it does

Pick a layer in the sidebar and the map recolors:

- **Priority (recommended)** — the headline ranking, using a blended heat measure.
- **Confidence (Monte Carlo)** — how often each tract is a top pick across 5,000 randomized model runs.
- **Heat** — daytime surface, nighttime surface, or air temperature.
- **Risk (heat + vulnerability + AC access)**, plus the raw **poverty** and **no-vehicle** components.
- **Distance to relief** — kilometers to the nearest Cool Zone or water point.

Air-conditioning access is used as an input to the score, not shown as its own layer.

Existing Cool Zones appear as markers, and a table lists the top-ranked tracts for whatever layer is selected.

## Key finding

After folding in air-conditioning access, the robust core holds: all five original top tracts stay in the top 20 in 88–100% of 5,000 randomized runs, and El Cajon 165.04 and Bostonia 165.02 are selected in every run. The high-confidence set (selected ≥88% of the time) is Bostonia 165.02, El Cajon 165.04, El Cajon 156.01, Santee 167.04, El Cajon 157.01, and Escondido 206.02 — inland East County communities that are hot day and night, populated, and relatively far from existing relief. Accounting for who can least cope also surfaces lower-AC candidates such as National City 119.02 and Vista 192.05.

## Data sources

- U.S. Census Bureau — 2022 cartographic tract and place boundaries; American Community Survey 2020-2024 (poverty, vehicle access, population) via the Census API.
- CDC / ATSDR — Social Vulnerability Index 2022.
- Google Earth Engine — Landsat 8/9 Collection-2 Level-2 (daytime land-surface temperature), MODIS MOD11A2 (nighttime land-surface temperature), Daymet V4 (modeled air temperature).
- San Diego County HHSA — Cool Zones 2025 (ArcGIS feature service).
- OpenStreetMap — drinking-water points; public libraries (used in a sensitivity check).
- Shade SD / San Diego Heat Risk Explorer (Southern California Extreme Heat Research Hub & San Diego Regional Climate Collaborative, Univ. of San Diego) — air-conditioning access, used under CC BY 4.0.

## Method (short version)

1. Rank each input to a 0-to-1 percentile across the county.
2. **Composite risk** = heat + social vulnerability + low air-conditioning access (who can least cope).
3. **Service gap** = distance to the nearest Cool Zone or water point.
4. **Priority** = risk x service gap x population, so a tract must be vulnerable, underserved, and populated at once.
5. Robustness: sensitivity analysis (weights and service definition), an independent-data logistic regression, multi-metric heat comparison, and a 5,000-run Monte Carlo that also varies the air-conditioning weight. See the methodology log for details and references.

## Files

- `app.py` — the Streamlit + Folium map application.
- `requirements.txt` — Python dependencies.
- `sd_tracts.geojson` — pre-computed tract data (scores, temperatures, priority, Monte Carlo confidence).
- `coolzones.geojson` — the 72 Cool Zone locations.
- *(`sd_app.geojson` is an older data file no longer used by the app; safe to delete.)*

## Run it locally

```
pip install -r requirements.txt
streamlit run app.py
```

The app reads the two GeoJSON files, so keep them in the same folder as `app.py`. The deployed app needs no API keys.

## Reproducing the data

The GeoJSON files are generated by a Google Colab notebook that pulls the sources above, builds the scores, and exports the map data. Regenerating it requires a free Census API key and a free Google Earth Engine (noncommercial) account.

## Limitations

- Land-surface temperature is not air temperature; the app reports day, night, air, and a blended measure, and the recommendations are ~80% stable across them.
- OpenStreetMap coverage is crowdsourced and not exhaustive; Cool Zones are seasonal and vary in hours and capacity.
- The Monte Carlo varied modeling choices (weight, heat metric), not the data's own margins of error.
- This is a screening tool that flags candidate underserved areas, not a proven optimal site.
